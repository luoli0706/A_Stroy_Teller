# 功能演进历史 (Functional History)

## [v0.1.0-alpha.2.1] - 2026-04-02
### 角色系统升级 (Role System)
- **从角色到演员**：角色不再受限于某个特定故事，而是拥有独立的人格、外貌与语言风格。这使得他们可以作为“演员”入驻任何故事框架。
- **角色扮演能力**：建立了角色扮演的逻辑框架，角色现在可以根据自身性格在不同故事中“扮演”对应的槽位。

---

## [v0.1.0-alpha.2] - 2026-04-02
### 体验与交互 (Experience)
- **界面模块化**：UI 界面更加整洁，日志查看与故事生成控制分离，提升了操作直观性。
- **结构化质检反馈**：质检结果现以结构化形式呈现（含分数、具体冲突点与修正建议），不再是简单的 PASS/FAIL。

### 创作深度 (Narrative Depth)
- **动态记忆演进**：系统现在会自动将每次生成的精彩片段浓缩并追加到角色的长期记忆汇总中，实现了“边写边记”。
- **人机协作雏形**：确立了大纲审核流程，为后续版本中用户在线干预生成方向提供了架构支持。

---

## [v0.1.0-alpha.1] - 2026-04-02
### 创作流优化 (Story Flow)
- **多视角并行**：系统支持多名角色同时撰写各自的视角，从逻辑上确保了视角生成的相对独立性与公平性。
- **高频 RAG 召回**：角色在叙述时能根据 `story_id` 精准回忆起同一故事框架下的过往章节或共享事实。
- **章节化管理**：生成的角色产出按照 `<story_id>__chapter_<timestamp>` 规则自动分类存储，支持连载与分块化创作。

### 用户体验 (User Experience)
- **毫秒级检索**：RAG 召回从之前的秒级降低至毫秒级，减少了生成前的预处理等待。
- **流式反馈**：UI 支持实时的节点状态日志与 Token 流式展示，生成过程更具透明度。
- **多语言支持**：初步建立了 I18N 体系，支持中英双语切换。
