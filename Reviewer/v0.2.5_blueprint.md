# A Story Teller v0.2.5 蓝图计划：可视化持久化与 UI 全面焕新

## 1. 版本目标
v0.2.5 的核心目标是将 v0.2.4 引入的“隐形持久化”能力显性化。通过新增历史回溯接口与现代化 UI 组件，用户可以直观地查看、对比并导出每一个生成断点的具体内容。

## 2. 核心任务清单

### 2.1 状态历史接口 (State History API)
- [ ] **接口开发**：在 `app/runtime.py` 中新增 `get_thread_history_async` 方法。
- [ ] **内容解析**：提取历史快照中的关键字段（大纲、适配身份、各角色视角、整合正文）。
- [ ] **顺序与归档**：按照 `step` 和 `created_at` 顺序排列，支持按节点名称（如 `finalize_output`）过滤。

### 2.2 UI 层现代化重构 (Modern UI Update)
- [ ] **历史侧边栏 (History Sidebar)**：新增一个可折叠的侧边栏，列出当前 `thread_id` 下的所有成功节点快照。
- [ ] **对比视图 (Checkpoint Viewer)**：点击历史记录时，右侧展示该时间点的故事内容快照。
- [ ] **样式统一化**：升级全局 UI 主题，使用更专业的配色方案，并增加 Markdown 预览支持。

### 2.3 流程增强
- [ ] **手动回滚点**：初步支持“从特定历史节点重启执行”的能力（Time Travel）。

## 3. 技术路线
1. **持久化层**：继续使用 `SqliteSaver` 提供的 `get_state_history` 方法。
2. **UI 逻辑**：将 `flet_app.py` 进一步组件化，引入 `HistoryPanel` 和 `StateDetailView` 组件。

## 4. 评估
本方案在技术上完全可行，且能显著提升“长篇生成”场景下的用户掌控感。它不仅是 UI 的更新，更是将“状态机”思维引入用户交互的关键一步。
